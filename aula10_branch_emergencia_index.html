aula10 - Criando branch de emergência

Para criar uma branch de emergência no Git, você pode seguir os
passos abaixo. O termo "emergência" geralmente indica que você
precisa interromper o que está fazendo para corrigir um bug crítico
em produção.



1. Certifique-se de estar na branch correta (geralmente main ou master)



Antes de criar a branch, atualize sua branch principal para ter
certeza de que está partindo da versão mais recente:

bash

git checkout main
git pull origin main



2. Crie a branch de emergência



Use um nome descritivo, como hotfix/titulo-do-bug ou emergency/correcao-critica:

bash

git checkout -b hotfix/titulo-do-bug

Ou, se preferir criar sem já mudar para ela:

bash

git branch hotfix/correcao-login
git checkout hotfix/correcao-login



3. Faça as alterações necessárias



Corrija o bug, adicione os arquivos modificados e faça o commit:

bash

git add .
git commit -m "hotfix: corrige bug crítico no login"



4. Envie a branch para o repositório remoto (opcional, mas recomendado)



Se desejar, envie a branch para o repositório remoto:

bash

git push origin hotfix/correcao-login



5. Depois de testar, faça o merge



Quando a correção estiver pronta, volte para a branch principal e faça o merge:

bash

git checkout main
git merge hotfix/correcao-login
git push origin main



6. (Opcional) Delete a branch de emergência



Apague a branch de emergência:

bash

git branch -D hotfix/correcao-login
git push origin --delete hotfix/correcao-login
