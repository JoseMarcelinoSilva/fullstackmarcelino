aula9 - Entendendo branchs

Entender branches (ramificações) é o passo mais importante para
dominar o Git. Elas são o coração do fluxo de trabalho em equipe e
da organização do código.

Vamos dividir isso em 3 níveis: Conceito, Comandos Práticos e Fluxo
de Trabalho.

1. O Conceito (A "Linha do Tempo" Paralela)

Imagine que seu código é um livro de história.

  • A main (ou master) é a versão oficial, a história principal que está publicada.
  • Uma branch é como criar um rascunho alternativo. Você pega o livro no capítulo 10 e decide escrever um final alternativo sem rasurar o livro original.

Por que usar?

  • Isolamento: Você pode testar ideias loucas sem quebrar o sistema em produção.
  • Paralelismo: Enquanto você faz a tela de login, seu colega faz a tela de cadastro na branch dele.
  • Segurança: O código só vai para a main depois que for revisado e testado.

2. A Mecânica (Como o Git funciona por baixo)

  • Ponteiro: Uma branch é apenas um ponteiro móvel (um adesivo) que aponta para um commit específico.
  • HEAD: O HEAD é um ponteiro especial que diz: "Em qual branch eu estou agora?".
  • Criação: Quando você cria uma branch, você está criando uma cópia do estado atual do código naquele exato momento.
