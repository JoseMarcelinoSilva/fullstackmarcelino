aula16 - Saber diferença entre commits

Para saber a diferença entre commits no Git, você tem várias opções,
dependendo do que exatamente quer comparar:



1. Comparar dois commits específicos (pelo hash)


git diff <hash-commit-A> <hash-commit-B>



2. Desfazer commits mantendo as alterações (soft reset)



bash

git reset --soft <hash-do-commit>

Exemplo:

git git diff abc123 def456



2. Comparar um commit com o estado atual (working directory)



git diff <hash-commit>



3. Comparar um commit com o commit pai (o que mudou naquele commit)



git show <hash-commit> ^

ou git diff <hash-commit>^ <hash-commit>



4. Ver a diferença entre dois commits de forma resumida (apenas nomes de arquivos)



git diff --name-only <hash-A> &hash-B>



5. Comparar o último commit com o anterior



git diff HEAD~1 HEAD



6. Ver a diferença entre dois branches (últimos commits de cada)

git diff <branch-A> <branch-B>


7. Ver o que mudou em um commit específico (mais detalhado que o show)

git log -p -1 <hash-commit>



8. Comparar dois commits ignorando espaços em branco

git diff -w <hash-A> <hash-B>



9. Para ver a diferença entre commits de forma visual (gráfica)

git difftool <hash-A> <hash-B>

(Isso abrirá uma ferramenta como Meld, Beyond Compare, etc., se configurada)



10. Ver a diferença entre o último commit e o commit de dois passos atrás



git diff HEAD~2 HEAD



Dica extra: facilitar a navegação



Use git log --oneline para ver os hashes de forma resumida e escolher quais commits comparar.

git log --oneline

Exemplo de saída:

text

abc1234 (HEAD -> main) Corrige bug no login
def5678 Adiciona nova funcionalidade de login
ghi9012 Atualiza documentação para o v1.0

Depois:

git diff abc1234 def5678
