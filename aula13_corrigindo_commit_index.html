aula13 - Corrigindo mensagem de commit

Para corrigir a última mensagem de commit no Git, você pode usar o comando:

bash

git commit --amend -m "Nova mensagem corrigida"

Isso substitui a mensagem do commit mais recente.



Se o commit já foi enviado ao repositório remoto (push)



Se você já fez push, será necessário forçar o envio:

bash

git push --force-with-lease

⚠️ Cuidado: só faça isso se ninguém mais tiver puxado (pull) esse
commit. Se outras pessoas já baixaram, isso pode causar conflitos.



Se você quer corrigir mensagens de commits mais antigos



Use rebase interativo:

bash

git rebase -i HEAD~3 # para os últimos 3 commits

No editor, troque pick por reword no commit que deseja corrigir, salve e edite a mensagem.



Se a mensagem tem erro de digitação e você quer algo mais rápido



bash

git commit --amend -c HEAD

Isso abre o editor para você alterar a mensagem.
