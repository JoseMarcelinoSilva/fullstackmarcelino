aula8 - Pedindo ajuda ao git

git commit -help

O comando git commit -help (na verdade, o correto é git commit
--help ou git help commit) é usado para exibir o manual de ajuda do
comando git commit diretamente no terminal.

Aqui está como ele funciona e o que acontece:

1. A diferença entre -h e --help

  • git commit -h → Exibe um resumo curto e rápido com as opções mais comuns (uso geral).
  • git commit --help → Abre o manual completo (geralmente no navegador ou no programa less), com explicações detalhadas, exemplos e todas as opções disponíveis.

OBS: -help (com um hífen só) não funciona. O correto é --help (dois hífens) ou -h.

2. O que é exibido

Quando você usa git commit --help, o Git:

  1. Localiza o arquivo de ajuda interno.
  2. Exibe no terminal (usando o paginador less por padrão) ou abre no navegador (dependendo da configuração).
  3. Mostra:
    • Nome do comando
    • Sinopse (como usar)
    • Descrição detalhada
    • Lista completa de opções (flags)
    • Exemplos práticos
    • Arquivos relacionados (como .gitignore)

3. Exemplo prático

# Ajuda resumida (recomendado para consultas rápidas) git commit -h
# Ajuda completa (manual extenso) git commit --help

4. Navegando no manual

Se o manual abrir no less (terminal), você pode:

  • Setas ou Page Up/Down para rolar
  • / para buscar uma palavra
  • q para sair

5. Alternativas

# Outra forma de ver a ajuda git help commit

Dica: Sempre que tiver dúvida sobre um comando Git, use --help ou -h
— é a documentação oficial e confiável!

Obs: para saber todos os comando que tem no git, use o comando git help --all
