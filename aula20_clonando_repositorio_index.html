O que é um conflito de merge?


Um conflito acontece quando duas branches alteraram as mesmas linhas
do mesmo arquivo de formas diferentes. O Git não sabe qual versão
manter e pede sua ajuda.



Como o Git te avisa sobre um conflito


$ git merge feature/nova-funcionalidade
Auto-merging arquivo.js
CONFLICT (content): Merge conflict in arquivo.js
Automatic merge failed; fix conflicts and then commit the result.



Identificando arquivos em conflito


Ver quais arquivos estão com conflito no merge:
git status

# Saída típica:
# both modified: arquivo.js
# both modified: style.css



Estrutura de um arquivo em conflito


O Git insere marcadores no arquivo:
function calcularPreco(produto) {
<<<<<<< HEAD
// Código da branch MAIN
return produto.preco * 1.10;
=======
// Código da branch FEATURE
return produto.preco * 1.15;
>>>>>>> feature/nova-funcionalidade
}



O que cada parte significa:

  • <<<<<<< HEAD — começo do que está na branch atual (main/master)
  • ======= — separador entre as duas versões
  • >>>>>>> feature/nova-funcionalidade — fim do que está na outra branch
