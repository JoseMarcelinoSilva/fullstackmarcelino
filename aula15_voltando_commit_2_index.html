aula15 - Voltando para outro commit - 2

Para voltar para outro commit no Git, você tem algumas opções,
dependendo do que exatamente deseja fazer:



1. Apenas visualizar o commit (não modifica nada)


bash

git checkout <hash-do-commit>

Isso coloca você em "detached HEAD" (cabeça desanexada). Você
pode explorar o código, mas qualquer commit novo criado aqui
ficará "perdido" se não criar um branch.

Para sair e voltar ao branch principal:

bash

git checkout main
# ou
git checkout master



2. Desfazer commits mantendo as alterações (soft reset)



bash

git reset --soft <hash-do-commit>

  • Mantém todas as alterações na staging area (prontas para commit)
  • Os commits a partir deste ponto são desfeitos
  • As mudanças permanecem no código



3. Desfazer commits mas manter alterações na pasta de trabalho (mixed reset)



bash

git reset <hash-do-commit>
# ou
git reset --mixed <hash-do-commit>

  • Mantém as alterações apenas na pasta de trabalho (working directory)
  • A staging area fica limpa
  • É o comportamento padrão do git reset



4. Desfazer commits e descartar todas as alterações (hard reset) ⚠️



bash

git reset --hard <hash-do-commit>

  • PERMANENTEMENTE descarta todos os commits e alterações após o hash informado
  • O código volta exatamente como estava naquele commit
  • ⚠️ CUIDADO: alterações não salvas serão perdidas!



5. Reverter commit mantendo histórico (safe)



bash

git revert <hash-do-commit>

  • Cria um NOVO commit que desfaz as alterações do commit especificado
  • Mantém o histórico intacto
  • É a opção mais segura, especialmente para commits já enviados ao remoto



Como saber o hash do commit?



bash

git log
# ou
git log --oneline

Exemplo de saída:

text

a1b2c3d (HEAD -> main) Último commit
e4f5g6h Commit anterior do HEAD
i7j8k9l Commit mais antigo



git stach

O git stash é como um "bolso mágico" onde você pode guardar
temporariamente as alterações não commitadas para limpar seu
diretório de trabalho.


📦 O que é e quando usar?

O stash é útil quando:



  • - Você precisa mudar de branch urgentemente mas não quer commitar código incompleto
  • - Quer testar algo rápido sem perder suas alterações atuais
  • - Precisa puxar atualizações (pull) mas tem modificações locais conflitantes
